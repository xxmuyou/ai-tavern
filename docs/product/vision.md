# 产品愿景

> 这份文档定义"我们在做什么、为谁做、为什么这样做"。具体玩法见 [`gameplay.md`](./gameplay.md)，日常生活模拟见 [`daily-life-sim.md`](./daily-life-sim.md)，v1 具体内容见 [`content.md`](./content.md)。

---

## 1. 产品类型

**AI 日常生活关系模拟 RPG**（AI Relationship Life Sim RPG）。

不是纯 AI 聊天玩具，不是传统恋爱手游，也不是 3D 生活模拟器。

核心是"日常生活感 × 关系推进 × 角色陪伴"：companion 不是静态聊天对象，而是生活在虚拟都市里的重要 NPC。用户通过每天拜访、活动、对话、事件、约会与回忆，逐步建立友谊、信任、暧昧、恋爱或疏离等长期关系。

**对标方向：**
- 借鉴 The Sims / inZOI 的生活感、角色状态、关系成长和日常事件
- 不复制其 3D 捏脸、建模、建造、开放地图、复杂职业系统
- 用 Web 端可承载的插图、AI 对话、规则事件、关系数值和回忆相册完成轻量生活模拟

## 2. 一句话定位

> 在虚拟现代都市里，每天拜访不同 AI companion，通过活动、对话、约会、事件和回忆推进多维度关系 —— 不做宏大 3D 世界，只做足够真实、持续、有付费价值的 AI 关系生活模拟。

## 3. 目标用户

**不限定性别，面向所有人。**

- 角色库需包含男性、女性、非二元等不同角色
- 用户自由选择想要互动的对象
- UI 与文案不默认假设用户性别（不出现"她喜欢你"这种暗示用户是男性的表达）

**目标市场：海外英文为主。**

- 产品 UI、官方角色对话、文案全部以英文为主
- 数据结构、prompt 模板、文本处理基于英文设计
- 中文等其他语言留给 v2+

## 4. 世界观

**现代都市 / 生活片。**

| 项 | 设定 |
|----|------|
| 时空 | 当代现代都市，**虚构城市**（不锁定真实地理） |
| 场景 | 咖啡馆、办公室、公寓、酒吧、街道、公园、健身房等日常都市场景 |
| 角色定位 | 同事、邻居、朋友、暧昧对象、陌生人、家人等现代都市人设 |
| 美术基调 | 动漫 / 二次元（anime / manga 风格） |

**明确不做：** 奇幻、科幻、历史、架空世界观。

## 5. 用户角色：隐形主角

用户在故事中是默认的"你"。

- **不让用户创建主角：** 不输入主角的外貌、姓名、性别、背景
- **第二人称叙述：** 所有 UI 与对话使用"you / your"
- **焦点全在 NPC：** 内容创作精力集中在 companion 身上

**为什么这样：** 降低创建门槛、最大化代入感、避免"用户想象 vs 系统呈现"的不一致。

类似 otome 经典设定 / Replika 风格。

## 6. 体验结构：开放日常

**没有"通关"，没有明确结局。**

- 游戏体验围绕真实时间中的每日状态展开，而不是固定章节路径
- 关系、事件、活动、约会与回忆随时间持续涌现
- 用户可以一直玩下去，关系可以深入到任意深度
- 类似 Replika（关系陪伴） + Character.ai（多角色） + 生活模拟器（事件感）的混合

**这不意味着没有节奏感：** 系统通过每日状态、活动目标、事件触发、关系阈值、约会邀请和里程碑回忆制造"剧情感"，但不强加线性主线。

**关于"章节"与剧情拍：** 产品不做全局主线通关章节，也不把用户锁进线性选项游戏。当前 v1.x 路线增加的是轻量 companion story beats：官方角色可有 authored arcs，自创角色可用剧情包 / 自写 / AI 辅助生成 3-5 拍 arc。它们提供“下一步做什么”的方向感，不替代开放日常和自由聊天。

## 7. 核心循环

```
打开 app
  ↓
看到 Today in Aurelia（日常主页：当前时间段、角色状态、推荐拜访）
  ↓
选择想见的 companion 或想去的场景
  ↓
拜访场景，看到 companion 今日位置 / 心情 / 正在做的事
  ↓
选择活动（聊天 / hang out / 邀请约会 / 送礼 / 修复紧张关系）
  ↓
AI 对话 + 事件 + 选择推动关系奇点变化
  ↓
关系阶段推进：新活动 / 新约会 / 新场景 / 里程碑回忆 / 关键 CG
  ↓
回到日常主页，期待下一个真实时间段或明天继续
```

具体机制见 [`gameplay.md`](./gameplay.md) 与 [`daily-life-sim.md`](./daily-life-sim.md)。

## 8. 角色系统：双轨

**官方精品角色（v1 提供 8-10 个）：**
- 由产品团队精心设计的角色卡（背景、人设、说话风格、关系定位）
- 体验质量基线 —— 新用户开箱即用就能获得好的内容
- 服务于产品的核心叙事调性

**用户自创角色：**
- 用户可以创建自己的角色（保留并简化现有 Guest 创建流程）
- 自由度高，个性化强
- 可在创建后设置剧情线：剧情包、用户自写或 AI 辅助草稿
- 默认私人体验，不进入官方推荐池；若后续开放公开角色发布，story arcs 默认私有、可选择共享只读线

**为什么双轨：** 官方保证质量与一致性，用户创建保证灵活性与长尾。两条路径共用数据模型与对话引擎。

## 9. AI 生成边界

为了平衡**质量 × 成本 × 灵活性**，明确区分预写与 AI 实时生成：

| 内容 | 来源 |
|------|------|
| 角色卡（背景、人设、说话风格） | 预写（官方）/ 用户填写（自创） |
| 场景定义（氛围、可能事件类型） | 预写 |
| 日常状态规则（时间段、出没、活动可用性） | 预写规则 + 现有 companion / scene 数据 |
| 关系阶段 / 活动目标 | 预写规则 |
| Companion story beats | 官方预写 / 剧情包模板 / 用户自写 / AI 辅助草稿 |
| 对话内容与回应 | AI 实时生成 |
| 角色对场景 / 事件的反应 | AI 实时生成 |
| 角色今日状态的文案表达 | AI 实时生成（规则决定状态，AI 只写表现） |
| 事件细节与情节发展 | AI 实时生成（在预定义的事件类型框架内） |
| 回忆日记摘要 | AI 辅助生成 + 结构化存档 |
| 关系数值变化规则 | 预写（规则引擎，非 AI） |

**核心原则：** AI 在被预写设定约束的框架内自由生成。规则决定"发生什么"，AI 负责"如何表达"。预写保证基线质量与一致性，AI 保证内容丰富与个性化。

## 10. 付费模式

**免费 + 额度限制 / 订阅去除限制。**

- **免费用户：** 每日有对话次数或 AI 调用上限
- **订阅用户：** 通过 Stripe 订阅去除限制

**v1 必须做到：**
- 免费即可完整体验产品（不是 demo wall）
- 订阅有明确的"去除限制"价值
- Stripe 集成走通

具体额度数值、订阅价格、计量方式见 [`monetization.md`](./monetization.md)。

## 12. 明确不做（v1 边界）

- ❌ 主角自定义（隐形主角不变）
- ❌ 3D 建模 / 捏脸 / 换装 / 房屋建造
- ❌ 复杂开放地图或实时城市模拟
- ❌ 奇幻 / 科幻 / 架空世界
- ❌ 线性预设主线（开放日常优先；轻量 companion story beats 不等于主线通关）
- ❌ 社区角色分享 / 公开浏览（v1 私密体验为先；后续公开发布时再接 story arc 共享）
- ❌ 中文等多语言（英文先打磨好）
- ❌ 战斗、解谜、复杂玩法（聚焦关系与故事）
- ❌ 自动翻译用户输入
- ❌ 成人露骨内容（v1 控制在 PG-13：暧昧、牵手、拥抱、告白、情绪亲密）
- ❌ 手绘里程碑 CG（v1 用程序合成；手绘留给 v2+ 收藏付费）
- ❌ Streak 连续登录奖励、email 推送、companion 主动私信（与陪伴调性不一致或复杂度过高，留给 v1.x）

## 13. 北极星

> 用户每天或每周打开 app 时，想知道某个 companion 今天在哪里、心情如何，以及能不能继续推进昨天那段关系。

如果做到了，产品就成立了。如果做不到，再多功能都没意义。
